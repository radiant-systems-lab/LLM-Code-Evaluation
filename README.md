# LLM Code Reproducibility: An Empirical Study of 300 AI-Generated Projects

**ACM REP '26 Artifact Submission**

## Abstract

We evaluate the reproducibility of code generated by three leading AI coding agents -- Claude Code, GitHub Copilot (Codex), and Gemini Code Assist -- across 300 projects in Python, Java, and JavaScript. Each project is tested in a clean Docker environment using only the dependencies the agent declares. Through three-layer dependency analysis (claimed, working, runtime), we find that only **68.3%** of projects execute successfully. Success rates vary dramatically by language (Python: 89.2%, JavaScript: 61.9%, Java: 44.0%) and agent (Claude: 73%, Gemini: 72%, Copilot: 60%). Transitive dependencies create a **4.3x explosion for Python**, **10.4x for Java**, and **31.4x for JavaScript** between what agents declare and what is actually installed.

## Key Results

| Metric | Value |
|--------|-------|
| Total projects | 300 (100 per agent x 3 languages) |

## Study Design

### Agents Evaluated
| Agent | Version | Access Method |
|-------|---------|---------------|
| Claude Code | Claude 3.5 Sonnet | CLI (`claude` command) |
| GitHub Copilot | Codex | VS Code extension |
| Gemini Code Assist | Gemini Pro | Google Cloud IDE |

### Prompt Distribution
| Language | Prompts | Count | Examples |
|----------|---------|-------|----------|
| Python | p_1 - p_40 | 40 | Web scraping, ML, APIs, NLP, data ETL |
| JavaScript | p_41 - p_75 | 35 | Express APIs, WebSocket, React, file I/O |
| Java | p_76 - p_100 | 25 | Spring Boot, Kafka, JPA, batch processing |

### Evaluation Methodology

Each of the 300 generated projects was tested in **isolated Docker containers**:

1. **Python**: `python:3.10-slim` container, `pip install -r requirements.txt`, `pip list --format=freeze`
2. **JavaScript**: `node:18-slim` container, `npm install`, `npm list --all --parseable`
3. **Java**: `maven:3.9-eclipse-temurin-17` container, `mvn dependency:tree`

Every container starts clean with zero cached dependencies, ensuring accurate transitive dependency counts.

## Data Description

### Raw CSVs (`data/raw_csvs/`)

Nine CSV files, one per agent-language combination. Each row represents one project with 20 columns:

| Column | Description |
|--------|-------------|
| `prompt_id` | Prompt identifier (p_1 to p_100) |
| `llm_name` | Agent name (claude, codex, gemini) |
| `language` | Programming language |
| `claimed_deps` | Dependencies declared by the agent |
| `claimed_count` | Number of claimed dependencies |
| `working_deps` | Dependencies after manual fixes |
| `working_count` | Number of working dependencies |
| `runtime_count` | Total packages installed (transitive) |
| `execution_success` | true / partial / false |
| `error_type` | Error classification |
| `completeness_gap` | Number of missing dependencies |
| `runtime_gap` | runtime_count - claimed_count |
